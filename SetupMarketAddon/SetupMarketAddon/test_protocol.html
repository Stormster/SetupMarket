Protocol Explanation

The setupmarket:// protocol should:

  1. Extract the setup ID from the URL (e.g., 12345 from setupmarket://setup/12345)
  2. Convert it to a Content Manager compatible URL: acmanager://setup/12345
  3. Launch Content Manager with the setup

If nothing happens when you click a test link, please check the following:

  1. Make sure the protocol is registered correctly (run register_protocol_fixed.bat)
  2. Check if Python is installed and in your PATH
  3. Check the log file at %USERPROFILE%\setupmarket_debug.log for errors

Technical Details

The protocol registration adds these entries to the Windows registry:

HKCU\SOFTWARE\Classes\setupmarket
HKCU\SOFTWARE\Classes\setupmarket\shell\open\command

The command entry points to a Python script that:

  1. Parses the setupmarket:// URL to extract the setup ID
  2. Converts it to an acmanager:// URL
  3. Logs all actions to %USERPROFILE%\setupmarket_debug.log
